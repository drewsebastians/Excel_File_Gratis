# Facts and Evidence Rules

## Evidence hierarchy

Gunakan urutan berikut:

1. Workbook aktual atau source file.
2. Script QA atau automated validation.
3. Screenshot atau visual-review artifact.
4. Frontmatter dan content schema.
5. Dokumentasi repository yang masih current.
6. Isi lama, hanya jika tidak bertentangan dengan bukti lain.

## Claims requiring verification

Verifikasi sebelum menulis:

- Jumlah dan nama sheet.
- Formula yang digunakan.
- Macro atau non-macro.
- Dashboard otomatis.
- Dropdown.
- Proteksi sheet.
- Maksimum baris.
- Kompatibilitas Google Sheets.
- Kompatibilitas versi Excel.
- File size.
- Nama file.
- Chart dan visual.
- Hubungan antar-sheet.
- Input cell color.
- Fitur filter, sort, pivot, query, atau automation.

## Unsupported claim handling

Jika klaim belum terbukti:

- Jangan masukkan ke body final.
- Masukkan ke `items needing owner verification`.
- Jangan mengubahnya menjadi kalimat samar untuk mempertahankan klaim.

## Examples and numbers

Contoh data harus:

- dinyatakan sebagai ilustrasi,
- konsisten dengan struktur workbook,
- menggunakan Rupiah atau format lokal jika relevan,
- tidak terlihat seperti data pengguna aktual,
- tidak menjanjikan hasil finansial atau bisnis.
