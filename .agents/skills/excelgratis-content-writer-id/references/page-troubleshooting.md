# Troubleshooting Page Playbook

## Primary intent

`problem solving`.

## Recommended structure

1. Gejala
2. Solusi tercepat
3. Penyebab paling umum
4. Langkah perbaikan
5. Cara memeriksa hasil
6. Solusi alternatif
7. Kapan perlu eskalasi

## Rules

- Tampilkan solusi paling mungkin sebelum teori panjang.
- Jangan membuat daftar penyebab tanpa urutan prioritas.
- Gunakan decision path bila penyebab berbeda.
- Jelaskan risiko sebelum langkah destruktif.
- Jangan menyarankan menghapus data, file, query, atau formula tanpa backup.
- Sebutkan versi atau environment bila memengaruhi solusi.
- Gunakan pesan error persis jika tersedia.

## Pattern

> Coba langkah ini terlebih dahulu: ...

Lalu:

- Jika berhasil, jelaskan mengapa.
- Jika belum, lanjutkan ke penyebab berikutnya.
